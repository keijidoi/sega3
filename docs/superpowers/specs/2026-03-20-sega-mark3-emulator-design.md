# Sega Mark III / Master System Emulator Design

## Overview

Flutter (Dart) で実装するセガ マークIII / Master System エミュレータ。
Android / iOS 両対応。学習目的と実用目的を兼ね備える。

## Requirements

- **プラットフォーム**: Android + iOS (Flutter)
- **実装言語**: エミュレーションコアを含め全て Dart で実装。ただし音声再生等のFlutterプラグイン（内部にネイティブコードを含む）は使用する。「カスタムネイティブコード（C/C++/FFI）は書かない」がポリシー。
- **ROM読み込み**: ボタンクリック → ファイルピッカー → 即起動
- **操作**: 画面上バーチャルパッドのみ（十字キー + ボタン1,2 + START）
- **セーブ**: ステートセーブ/ロード（3スロット）+ バッテリーバックアップ
- **スコープ**: 以下のゲームが起動・プレイ可能なレベルを目標とする:
  - Alex Kidd in Miracle World
  - Hang-On
  - Fantasy Zone
- **リージョン**: NTSC のみ（PAL は対象外。PAL: 313スキャンライン/50Hz、クロック3.546894MHz）
- **BIOS**: BIOSは非エミュレート。実行はカートリッジROMのアドレス $0000 から直接開始する。

## Architecture: Component-Separated

各ハードウェアコンポーネントを独立クラスとして実装し、メモリバス経由で通信する。

│  │ ボタン   │  │(CustomPainter)│ │  パッド  │  │
│  └──────────┘  └──────────┘  └───────────┘  │
└──────────────────┬──────────────────────────┘

│            Emulator (システム統合)            │
│  - メインループ (フレーム単位)                │

### Emulator クラスの役割

- `ChangeNotifier` を継承し、フレーム完了時に `notifyListeners()` でUI更新
- メインエミュレーションループは **メインIsolate** 上で `Ticker` を使用して駆動（`Timer.periodic` は使用しない — vsync同期のため）
- 1フレーム分の処理を `Ticker` コールバック内で同期実行
- メインループ内はオブジェクト生成を避け、GC圧力を最小化する

### MemoryBus インターフェース

```dart
abstract class Bus {

  void ioWrite(int port, int value);  // I/Oポート書き込み

```

## Component Details

### Z80 CPU

- **クロック**: 3.546895 MHz (NTSC)
- **レジスタ**: A,B,C,D,E,H,L,F (メイン+裏), IX, IY, SP, PC, I, R
- **フラグ**: S, Z, H, P/V, N, C
- **命令**: 全命令セット実装（約700命令、プレフィクス CB/DD/ED/FD 含む）
- **実行**: `step()` メソッドで1命令ずつ実行、消費サイクル数を返す
- **割り込み**:
  - NMI: ポーズボタン → 0x0066 にジャンプ
  - INT: VDP割り込み → IM1 で RST 0x38
- **実装**: 大きな switch 文でオペコードデコード、二段階デコード（プレフィクス）
- **サイクルテーブル**: 各命令の正確なサイクル数を定数テーブルで管理
- **メモリ**: `Uint8List` でパフォーマンス確保
- **整数マスキング**: 全演算で `& 0xFF`（8bit）/ `& 0xFFFF`（16bit）を徹底し、Dart の64bit整数からの溢れを防ぐ
- **不正オペコード**: 未定義オペコードはNOP扱い（サイクルは4消費）。デバッグビルドではコンソールに警告出力。

### VDP (Video Display Processor)

- **ベース**: TMS9918A 派生（Master System 拡張版）
- **VRAM**: 16KB
- **CRAM**: 32エントリ（BG 16色 + スプライト 16色、各6bit RGB）
- **主要モード**: Mode 4（ほぼ全ゲームで使用）
  - 8x8 タイルベース、横32×縦24タイル = 256x192 表示（標準）
  - 224ラインモード（32×28タイル）は少数ゲーム用。初期スコープでは非対応とする。
  - Mode 0-3 (TMS9918A互換) は最低限サポート
- **スキャンライン描画**:
  ```
  1フレーム = 262 スキャンライン (NTSC)
  ├─ 0-191: アクティブ描画
  ├─ 192: Vblank 割り込み発生（スキャンライン192の描画完了後）
  └─ 193-261: 垂直ブランク期間
  ```
- **割り込みタイミング精度**: スキャンライン粒度（サイクル精度は不要）。ターゲットゲームには十分。
- **各スキャンライン処理**:
  1. 背景タイル1ライン分レンダリング → ラインバッファ
  2. スプライト1ライン分レンダリング → 重ねる
  3. スプライトオーバーフロー検出（1ラインに9個以上）
  4. ライン割り込みカウンタ処理
- **スプライト**: 最大64個、1ライン最大8個、8x8 or 8x16、衝突検出
- **スクロール**: 水平・垂直、上2行/右8列固定オプション
- **出力**: `Uint32List` フレームバッファ (256x192 ARGB)

#### VDP 内部状態

- **アドレスレジスタ**: 14bit、書き込みペンディングラッチ（1バイト目/2バイト目トグル）
  - ステータスレジスタ読み取り時にラッチリセット
  - コントロールポート($BF)への書き込みで1バイト目→2バイト目→実行のシーケンス
- **読み取りバッファ**: VDPデータポート読み取りはバッファ経由（前回の読み取り値を返し、新しい値をプリフェッチ）
- **ステータスレジスタ** ($BF 読み取り):
  - bit 7: VBlank割り込みペンディング
  - bit 6: スプライトオーバーフロー
  - bit 5: スプライト衝突
  - bit 4-0: (Mode 4では未使用)
  - 読み取り時にbit 7, 6, 5がクリアされ、アドレスラッチもリセット

### PSG (SN76489)

- **チャンネル**: トーン×3（矩形波、10bitカウンタ）+ ノイズ×1
- **ボリューム**: 各チャンネル4bit（0=最大、15=無音）
- **出力**: CPUクロックから44100Hzサンプル生成、チャンネルミックス
- **音声再生**: Flutter音声プラグイン（`flutter_soloud` 等）でPCM再生
- **レジスタ**: bit7=1 ラッチ（チャンネル・タイプ選択+下位データ）、bit7=0 データ

#### ノイズチャンネル詳細

- **モード選択** (bit 2): 0 = 周期ノイズ、1 = ホワイトノイズ
- **シフトレート** (bit 1-0):
  - 00: クロック/512
  - 01: クロック/1024
  - 10: クロック/2048
  - 11: トーンチャンネル3の周波数を使用
- **LFSR**: 16bit、セガ独自タップ位置 bit 0 と bit 3（TIオリジナルとは異なる）
  - フィードバックビット = bit0 XOR bit3（ホワイトノイズ時）
  - フィードバックビット = bit0（周期ノイズ時）

### I/O Ports

| ポート | 読み | 書き |
|--------|------|------|
| $3E | - | メモリ制御（BIOS/カートリッジ有効化ビット） |
| $3F | I/Oポート状態 | ナショナライゼーション制御（リージョン検出） |
| $7E | VDP V counter | PSG |
| $7F | VDP H counter | PSG |
| $BE | VDP データ | VDP データ |
| $BF | VDP ステータス | VDP レジスタ/アドレス |
| $DC | コントローラー1 + コントローラー2一部 | - |
| $DD | コントローラー2 + リセット/ポーズ | - |

**ポート $3F**: 日本版/海外版の判定に使用。多くのゲームが起動時にこのポートを読み取る。エクスポート（海外）モードに固定して返す。

### Memory Map

```
$0000-$BFFF  カートリッジROM（バンク切り替え）
  $0000-$03FF  バンク切り替え無効領域（常にROM先頭）
  $0400-$3FFF  スロット0（$FFFD で選択）
  $4000-$7FFF  スロット1（$FFFE で選択）
  $8000-$BFFF  スロット2（$FFFF で選択、RAM切替可）
$C000-$DFFF  8KB システムRAM
$E000-$FFFF  RAMミラー
  $FFFC       RAM/ROM切替制御
  $FFFD-$FFFF バンクレジスタ（書き込み時）
```

**初期バンクレジスタ値（電源投入時）**: Slot 0 = 0, Slot 1 = 1, Slot 2 = 2。これが未設定だとゲームが起動しない。

**ROM空間への書き込み**: 無視（サイレント、エラーなし）。ただし $FFFC-$FFFF への書き込みはバンクレジスタとして処理（RAM空間 $C000-$DFFF へのミラー書き込み経由）。

### ROM ヘッダ解析・マッパー検出

- **ヘッダ位置**: $7FF0（8KB境界）、$3FF0、$1FF0 を順に検索
- **TMR SEGA マジック**: ヘッダの先頭8バイトで検出
- **512バイトコピアヘッダ**: ROMサイズが512バイト余剰の場合、先頭512バイトを除去
- **マッパー検出**:
  - **セガマッパー**（標準）: $FFFD-$FFFF でバンク切り替え
  - **コードマスターズマッパー**: $0000, $4000, $8000 でバンク切り替え（Micro Machines等）
  - ヘッダ情報とROMサイズから自動判定

## UI Design

### ホーム画面
- 中央に「ROM を読み込む」ボタン
- `file_picker` パッケージで .sms / .sg ファイル選択
- 選択後、即ゲーム画面へ遷移

### ゲーム画面
- 上部: ゲーム画面（4:3 アスペクト比、`CustomPainter`）
- 下部: バーチャルパッド
- メニューボタン（一時停止、ステートセーブ/ロード、戻る）

### バーチャルパッド
```
┌─────────────────────────────┐
│      ゲーム画面 (4:3)        │
│                             │
├─────────────────────────────┤
│   [↑]              [1] [2] │
│ [←][●][→]                  │
│   [↓]        [START]       │
└─────────────────────────────┘
```
- 十字キー: GestureDetector で8方向対応
- ボタン1, 2: マークIIIの2ボタン
- START: NMI 発生（ポーズ）

### 描画パイプライン

```
Emulator → Uint32List フレームバッファ → notifyListeners()
→ CustomPainter.paint() → ui.decodeImageFromPixels() → Canvas.drawImage()
```

**パフォーマンス最適化**:
- `ImmutableBuffer.fromUint8List()` + `ImageDescriptor` で同期的に `ui.Image` を生成（`decodeImageFromPixels` の非同期コスト回避）
- フレームバッファは事前確保した `Uint32List` を再利用（毎フレーム新規生成しない）
- `Ticker`（`SingleTickerProviderStateMixin`）で vsync 同期の60fps描画

## Save System

### ステートセーブ（どこでもセーブ）

**バイナリフォーマット:**
```
[4 bytes] マジック: "SMS\0"
[2 bytes] バージョン: 0x0001
[4 bytes] Z80 ステートオフセット
[4 bytes] VDP ステートオフセット
[4 bytes] PSG ステートオフセット
[4 bytes] メモリステートオフセット
--- Z80 State ---
全レジスタ、割り込み状態、サイクルカウンタ
--- VDP State ---
VRAM 16KB、CRAM 32bytes、レジスタ、内部状態（アドレスラッチ、読み取りバッファ等）
--- PSG State ---
チャンネル状態（周波数、ボリューム、LFSR）
--- Memory State ---
RAM 8KB、バンクレジスタ、カートリッジRAM（存在時）
```

- `path_provider` でアプリ内ディレクトリ保存
- 3スロット
- バージョン番号でフォーマット互換性管理

### バッテリーバックアップ
- スロット2がRAMモードの場合、RAM内容を自動保存
- ゲーム終了時・バックグラウンド移行時に書き出し
- ROM名をキーにファイル管理

## Dependencies

- `file_picker` — ROM ファイル選択
- `path_provider` — セーブデータ保存先
- `flutter_soloud` (または同等) — PCM音声再生（内部にネイティブコードを含むが、カスタムネイティブコードは不要）

## Timing

- **1フレーム**: 228 クロック × 262 スキャンライン = 59,736 クロック (NTSC)
- **フレームレート**: 約 59.92 fps
- **同期方式**: スキャンライン単位で CPU と VDP を交互に実行
  1. CPU を 228 サイクル分実行
  2. VDP で 1 スキャンライン描画
  3. 262回繰り返して1フレーム完成
  4. PSG は CPU 実行中にサンプル蓄積

## Error Handling

- **不正オペコード**: NOP扱い（4サイクル消費）、デバッグ時に警告出力
- **ROM破損/サイズ不正**: ファイル読み込み時にバリデーション、エラーダイアログ表示
- **ROM空間への書き込み**: サイレントに無視（バンクレジスタ以外）

## Testing Strategy

- Z80 個別命令ユニットテスト
- Z80 テストROM (zexall等) での統合テスト
- VDP レンダリングのスナップショットテスト
- PSG 波形生成テスト
- ターゲットゲーム3本の起動・動作確認
